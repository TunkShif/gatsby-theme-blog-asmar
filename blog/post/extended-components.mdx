# An Introduction to MDX

If you've ever used `React.js`, you must know what `JSX` is. It is a syntax extension to JavaScript, which allows you to write HTML tags in JavaScript.

To know more about JSX, check out the [React docs][0]!

```jsx
const title = <h1>Hello, JSX!</h1>
const link = <a href="#" className="link">click me</a>

const Page = (props) => {
  return (
    <article>
      <h1>Title</h1>
      <p>Oh my simple page!</p>
    </article>
  )
}
```

Similarly, MDX is a syntax extension for markdown, which allows you to use JSX in markdown!

So you can just import React components into MDX to use.

```jsx
import { Chart } from '../components/chart'

# Hereâ€™s a chart

The chart is rendered inside our MDX document.

<Chart />
```

To know more about MDX, check out the its [Homepage][1].

## A Starter MDX Template for Blog Post

On top of the mdx file, there's a section called `frontmatter` for page metadata in YAML syntax.

Be sure to fill in all these fields. And if you don't want any tags, just leave an empty array here. Do **NOT** just delete the `tags` field, or it will cause problems when rendering the page.

Below the `frontmatter` section, you can import any installed React components you wan to use in the page.

Please be **sure** to always add an empty line between two elements to avoid any potential markdown parsing related issues.

```jsx
---
title: Write a Title Here
date: 2021-07-22
desc: "guide to extended components for mdx"
tags: ["markdown", "guide"]
---

import { Chart } from '../components/chart'

# Heading

Content goes here...

Use imported components if you want.

<Chart />
```

Some components are auto imported for all mdx files, which will be introduced in this post. You don't need to import these components before using it.

## Image

Use zoomable `Image` components in mdx to get better image display.

Display the image in fullscreen by just clicking it. Click the image caption to get the origianl link.

Also you can pass a `url` prop to the component, then click the caption text will direct to this url.

Powered by `react-medium-image-zoom`.

```jsx
// used just the same way as <img> tag
<Image
  src="https://images.pexels.com/photos/167699/pexels-photo-167699.jpeg?auto=compress"
  alt="serenity forest" />

// optional props for width and height
<Image
  src="https://images.pexels.com/photos/167699/pexels-photo-167699.jpeg?auto=compress"
  alt="serenity forest"
  width={1260}
  height={750} />
```

However, this component can get ***messy*** when rendered in the page, take care when using it!

All components inside the `Column` must be `block` element, if you want to write markdown inside it, **wrap** it with a `div` tag.

And, be **sure** to leave an empty line after the opening tag (`<div>`) and before the closing tag (`</div>`), as shown below. Or the first line inside the `div` tag won't be rendered as markdown.

By default, the `Column` component won't show double column layout in mobile screen, but you can pass a `mobile` prop to the component to enable it.

The implementation of the `Column` component can be found [here][2].

